How Does Bulk Upload Work?
Powered by Orama
A full-text & vector search engine — running 100% in your browser. No backend. No server. Your files never leave your machine.

This isn't a simple pattern matcher (regex). Orama is a hybrid search engine that combines full-text search with vector-based semantic understanding — the same tech behind modern search engines.

How It Works
1. Index
Every dropped file is inserted into an Orama search database right in your browser. Filenames become searchable documents.
2. Search
Orama uses BM25 ranking to find the best match for each test slot — scoring relevance, not just checking exact letters.
3. Place
The highest-scoring matches are routed to the correct HW tab → Q subtab → test case. Missing tabs are created on the fly.
Why It's Smarter Than Regex

Regex only sees letters. Orama understands meaning. Even if your files aren't named exactly hw1q1in1, Orama's tokenizer and scoring engine will still figure out where they belong.

Approach How it sees hw1q2out3
Regex It's the letters h, w, 1, q, 2, o, u, t, 3. Exact match required — change one letter and it fails.
Orama (Text) It's a searchable document containing tokens: hw1q2out3. Stemming & BM25 ranking find the best match even with variations.
Orama (Vector) It's a point in meaning-space that's closest to "homework 1, question 2, output, test 3" — no exact naming required.
Recommended Naming
hwXqYinZ  or  hwXqYoutZ  —  X=homework, Y=question, in/out=type, Z=test case

Example: hw1q2out3.txt → Orama scores it 0.96 and routes it to HW1 · Q2 · Expected Output · Test 3. ✨
